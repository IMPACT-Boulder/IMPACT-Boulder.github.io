This is a beta version which probably has not been sufficiently tested.
  • Version 3.1.0 (Build 408):
    • Improves the overall performance and stability.
    • Fixes the selection of the anchor line via the line table.
    • Fixes the selection of ignored lines via the line table.
  • Version 3.1.0 (Build 407):
    • Improves the way auxiliary data downloaded from Impact are processed.
    • Estimates the impact time of downloaded data from the fitted target waveform.
  • Version 3.1.0 (Build 406):
    • Events associated with the wrong QD feature are noe marked in the Accelerator Statistics tool.
    • Fixes an issue introduced in the previous release where events could not any longer be selected in the Accelerator view.
    • Fixes an issue that affected the upload of accelerator waveforms.
  • Version 3.1.0 (Build 405):
    • Supports MacOS Ventura. Note that you need to install the update before you upgrade your computer to MacOS Ventura. Ventura's GateKeeper will not accept the previous version to run.
    • Fixes an issue that affected the undoing of stretch parameter changes.
    • Fixed plotting of spectra in the mass domain.
    • Fixed and improved the browser mode.
    • Fixed an issue that affected the change tracking of catalog items such as line and composition types
    • Improved the overall performance and stability
  • Version 3.1.0 (Build 403):
    • Provides a predefined DDA instrument.
    • Fixes an issue that affected the undoing of assigned anchor lines.
    • Fixes a rare issue that affected the import of scope data.
    • Other bug fixes.
  • Version 3.1.0 (Build 402):
    • The next few beta releases will only include bug fixes to prepare the release of the stable version 3.1.0.
    • Multiple behind the scene improvements and bug fixes.
  • Version 3.1.0 (Build 401):
    • Provides a predefined IDEX instrument.
    • The database query builder now has an approximatly equal operator for event time searches.
    • Coded reviewed and refactored all statistics tools.
    • A large number of bug fixes.
  • Version 3.1.0 (Build 399):
    • Moved Import and Export submenues from File menu to main bar.
    • Query builder time stamp format is now using the long form.
    • Fixes an issue which caused the app to crash after opening the query dialog when the current window has an open side bar.
    • Fixes an issue which caused the app to crash after closing a project when certain statistics views are still open.
    • Other bug fixes.
  • Version 3.1.0 (Build 398):
    • Database import is more forgiving in cas of data integrity issues.
    • Data base query alters are now presented.
    • Line labels are now correctly placed in browser mode.
    • Other bug fixes.
  • Version 3.1.0 (Build 397):
    • Fixes an issue where sometimes the cursor was not any longer responding to mouse movements.
    • Fixes some menu items in the main menu.
    • Rewrote data viewer code to increase stability.
    • Other bug fixes.
  • Version 3.1.0 (Build 395):
    • Queries of large data sets are now much faster.
    • The number of bins to compute the composition statistics can now be set by the user.
    • Rewrote data viewer code to increase stability.
    • Fixes an issue that prevented the line drop down menu in the Spectrum Tab's line table to be updated.
    • Multiple bug fixes.
  • Version 3.1.0 (Build 394):
    • Data sets can now be quality marked in the Project/General dialog.
    • Supports Impact database version 3.4.0, which now accepts QD (accelerator) data.
    • Ignored and locked state of mass lines are now exported to Impact DB.
    • H5 imports and exports now include QD (accelerator) data
    • Anchor lines are now correctly loaded from Impact DB.
    • Fixes an issue where CDA data sets could not be merged.
  • Version 3.1.0 (Build 393):
    • This version brings support of user-defined Python functions. Right now Python modules computing line shapes and waveform models can be called by Spectrum. Python plugins need to registered via the Preferences dialog. Because Spectrum is sandboxed, it cannot call call external programs including the Python interpreter. For this reason, a full Python 3.8 package including NumPy is bundled into the App (which explains the large increase of the app's size).
    • The H5 import and export capabilities have been further improved. H5 archives starting with a LILBID group are considered to contain LILBID raw data generated by the Freie University group and will be coverted into a Spectrum project.
    • This version now imports CCSDS packet data generated by SUDA FSW version 9.
    • Impproved import of PSU data. The correlation between PSU and QD data can be analysed indepth with the new QD Detection Statistics tool.
    • New tool for visualizing the impact plasma composition as function of the impact speed, impact energy, and impact energy density.
    • Improved Impact database import and export.
    • Multiple bug fixes.
  • Version 3.1.0 (Build 388):
    • New Impact Database account management, which moved to the Preferences dialog. Account credentials will be stored in the user's local keychain, implying that passwords only need entered at the time of the account's registration with the app. Accounts can be selected in the main menu's Impact menu.
    • Impact Database action menus have been moved to stand alone menu Impact located in the main menu bar.
    • The H5 export has been significantly enhanced to allow for the reimport of H5 files. Import of H5 Spectrum archives are now possible.
    • Major parts of the app's code base have been moved to external packages to improve the quality and testability of the software.
    • Many minor improvenments and bug fixes.
